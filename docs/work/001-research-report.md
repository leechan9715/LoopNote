## 1. 시장 현황 및 트렌드
### 1.1 에듀테크 시장의 변화
*   단순 문제 풀이(Drill & Kill) 중심에서 **학습 동기 부여** 및 **회복 탄력성(Resilience)** 중심으로 패러다임 전환.
*   '오답'을 실패가 아닌 '성장의 기회'로 리브랜딩하는 전략이 중요해짐.

### 1.2 초등 교육의 특성
*   **주의 집중 시간(Attention Span)**: 초등학생의 평균 집중 시간은 10~15분 내외. -> '10분 미션'은 심리학적으로 매우 적절한 목표 설정.
*   **자기 효능감**: 작은 성공 경험(Small Wins)의 반복이 장기적인 학습 습관을 결정함.

## 2. 경쟁 서비스 분석 (벤치마킹)
| 서비스명 | 핵심 전략 | 루프노트의 차별화 포인트 (Unfair Advantage) |
| :--- | :--- | :--- |
| **웅진 스마트올 / 밀크T** | 전과목 콘텐츠, 태블릿 기반 | 방대한 양 대신 '오답 해결'이라는 **특정 문제 상황(Pain Point)**에 집중. |

| **체크메이트 / 설탭** | 비대면 튜터링 | 사람 대신 **AI가 실시간 막힘 구간을 감지**하여 휴먼 터치에 준하는 가이드 제공. |

## 3. 기술적 실현 가능성 검토
### 3.1 AI 오답 분석 및 가이드
*   **OCR/Math Solver**: 학생이 푼 종이 문제지나 디지털 노트를 스캔하여 오답의 위치와 유형 파악 가능.
*   **LLM 기반 힌트 생성**: 정답을 바로 노출하지 않고, 학생의 사고 수준에 맞춘 '단계별 힌트(Socratic Tutoring)' 생성 기술 적용 가능.

### 3.2 10분 미션 설계 (Gamification)
*   **미션 알고리즘**: 오답의 난이도에 따라 10분 내에 완수 가능한 분량으로 문제를 쪼개는(Chunking) 기술 필요.

### 3.3 프론트엔드 및 아키텍처 전략
*   **Next.js 활용**: 빠른 인터랙션과 학습 콘텐츠의 SEO 최적화를 위해 Next.js(App Router)를 주력 프레임워크로 사용.
*   **서버사이드 렌더링(SSR/ISR)**: 초등학생 사용자의 기기 환경이 다양함을 고려하여, 서버사이드 최적화를 통한 저사양 기기에서의 빠른 로딩 보장.
*   **API Routes**: 별도의 백엔드 구축 전 MVP 단계에서 Next.js API Routes를 통해 AI 모델 연동 및 비즈니스 로직을 효율적으로 처리 가능.

## 4. 리서치 결론 및 제언
*   **핵심 가치**: "오답을 마주했을 때의 두려움을 10분의 즐거운 도전으로 바꾼다."
*   **성공 열쇠**: AI가 얼마나 정교하게 학생의 '막힘(Frustration)' 지점을 찾아내고, 좌절하지 않을 정도의 최적의 힌트를 주는가에 달림.
